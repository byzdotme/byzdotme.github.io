# 01 · 核心命令与高效工作流

> 目标：读完这篇，你 80% 的日常操作都能脱手完成。版本：截至 2026/04。

## 1. 心智模型：四种输入前缀

CC 输入框里前缀决定"走哪条路"：

| 前缀 | 作用 | 例子 |
|------|------|------|
| `/` | **斜杠命令 / Skill**：调用内建命令或 prompt 驱动的 skill | `/clear`、`/compact`、`/review`、`/batch` |
| `@` | **文件/目录引用**：把内容塞进上下文 | `@src/server.ts`、`@docs/` |
| `!` | **直接执行 shell**：不走 Agent 推理 | `!git status`、`!pnpm test` |
| 无前缀 | **自然语言任务**：交给 Agent | "帮我把 X 模块的日志改成结构化日志" |

> 小技巧：`!` 跑出来的结果会自动进入上下文，比让 Agent "跑一下 git status" 省一次工具调用往返。
>
> 官方已把"slash command"和"skill"合并：`/xxx` 既可能是硬编码命令，也可能是 prompt 驱动的 skill；旧 `commands/` 目录已弃用，新写法都用 skill（详见 02 篇）。

## 2. 真正每天都会用的斜杠命令

按使用频次排序，不是按字母序。

### 2.1 会话与上下文

| 命令 | 什么时候用 | 注意 |
|------|-----------|------|
| `/clear`（别名 `/reset`、`/new`） | **换一个新任务前必做**，清空当前上下文但保留 CLAUDE.md | 上下文是 CC 最贵的资源，别让上一个任务的垃圾污染下一个 |
| `/compact [可选指令]` | 任务还要继续但上下文快爆了 | 传指令可以指定保留什么，例如 `/compact 只保留对支付模块的决策` |
| `/resume`（别名 `/continue`） | 恢复一个历史会话（弹出选择器） | 跨天长任务；同一目录下所有会话都列出来 |
| `/branch [name]`（别名 `/fork`） | 从当前对话分叉一个新会话 | 想"如果当初约束不同会怎样"的 A/B 探索 |
| `/rewind`（别名 `/checkpoint`、`/undo`） | 回退当前会话（可选只回退对话、只回退代码、或两者） | 走错路时比 `git reset` 更稳，会话和工作区一起回滚 |
| `/rename [name]` | 给当前会话起名，方便后续 `/resume` | 长任务必备 |
| `/cost`（别名 `/usage`） | 看当前 session 花了多少 token / 钱 | 大改动前后各看一次，建立成本直觉 |
| `/model` | 切换底层模型 | 简单重构 Sonnet 4.6，复杂推理 Opus 4.7，批量小任务 Haiku 4.5 |
| `/effort [level\|auto]` | **新（2026）**：调思考预算 `low/medium/high/xhigh/max` | Haiku 不支持 effort；Opus 4.7 默认 xhigh；Sonnet 4.6 上限 max |
| `/fast [on\|off]` | **新（2026）**：用 Opus 4.6 + 跳过部分推理特效 | 简单任务显著降本；架构题别开 |
| `/status` / `/config`（别名 `/settings`） | 设置面板，看版本、模型、账户、output style | 切 output style、查配置文件来源都在这里 |
| `/context` | 可视化当前上下文占用 | 调上下文预算时第一时间看 |
| `/export [filename]` / `/copy [N]` | 导出对话 / 复制最后一段回答 | 写复盘时省力 |

> 说明：`--continue` / `-c`、`--resume` / `-r` 是 **CLI 启动 flag**（在 shell 里 `claude -c` / `claude -r` 用），不是会话内 slash 命令。会话里只有 `/resume`。

### 2.2 项目初始化 & 记忆

| 命令 | 作用 |
|------|------|
| `/init` | 让 CC 扫描当前仓库，自动生成一份 `CLAUDE.md` 草稿。**新仓库第一件事**。生成完一定要人工 review。设 `CLAUDE_CODE_NEW_INIT=1` 开交互式流程 |
| `/memory` | 打开 CLAUDE.md 编辑视图；同时是开关 **auto memory** 的入口（v2.1.59+） |
| `#` 开头的消息 | 快捷写入记忆：`# 我们团队用 pnpm 而不是 npm` 会追加到 CLAUDE.md（默认写到项目根；要写到子目录请用自然语言指定） |

### 2.3 编码 & 审查

| 命令 | 作用 |
|------|------|
| `/review [PR]` | 对当前改动或指定 PR 做 code review，PR 自查首选 |
| `/ultrareview [PR]` | **新（2026）**：云端多代理深度审查；比 `/review` 慢但更全；3 次免费 |
| `/diff` | 打开交互式 diff 查看器 |
| `!git diff` | 看 CC 本次工作区总改动（最稳） |
| `/rewind` | 回退本次会话——可选只回退对话、只回退代码、或两者 |
| `/agents` | 管理 subagents（子代理） |
| `/skills` | 列出当前可用所有 skills |
| `/tasks`（别名 `/bashes`） | 管理后台任务（Ctrl-B 之后的） |

> **进入 Plan 模式** 推荐 **Shift+Tab** 循环（详见 §4）。也可 `/plan [task description]` 直接传任务。

### 2.4 工具 & 权限

| 命令 | 作用 |
|------|------|
| `/permissions`（别名 `/allowed-tools`） | 查看/修改允许哪些工具、哪些 shell 命令自动通过 |
| `/hooks` | 查看已配置 hook |
| `/mcp` | 管理 MCP 服务器 |
| `/plugin` | 安装/管理 plugins（详见 02） |
| `/sandbox` | 切换沙箱模式 |
| `/security-review` | **新（2026）**：扫描当前分支安全问题 |
| `/help` | 列出所有可用命令 |
| `/doctor` | 诊断 CC 安装与设置 |

**Permission mode 共六档**（Shift+Tab 循环）：

| 模式 | 行为 |
|------|------|
| `default` | 危险动作弹窗确认 |
| `acceptEdits` | 自动接受文件编辑，但 bash 仍弹窗 |
| `plan` | 只读，产出计划，不改代码 |
| `auto` | **新**：让 CC 自己判断要不要弹窗（基于工具风险） |
| `dontAsk` | **新**：什么都不问、走默认决策；危险但不破墙 |
| `bypassPermissions`（= `--dangerously-skip-permissions`） | 全部跳过；本地玩可以，**生产/CI 严禁** |

### 2.5 Bundled Skills（2026 新增的"提示词驱动命令"）

这些标记为 `[Skill]`，**本质是官方维护的 prompt 模板**，不是硬编码逻辑：

| 命令 | 用途 | ROI 提示 |
|------|------|---------|
| `/batch <instruction>` | 自动派 5–30 个 worktree 子代理并行做大规模改动 | 跨包大改名 / 大批量迁移；省时间但费 token |
| `/debug [description]` | 启用 debug 日志并分析问题 | 卡了不知道为什么时第一招 |
| `/simplify [focus]` | 审查修改文件，找代码重用与效率问题 | 提交前自查重复实现 |
| `/loop [interval] [prompt]` | 重复运行 prompt（如 `/loop 5m`） | 盯 CI、轮询任务状态 |
| `/security-review` | 安全扫描分支 | 提 PR 前自查 |
| `/ultraplan <prompt>` | 浏览器中起草计划，可选执行位置 | 大型架构决策 |
| `/ultrareview [PR]` | 云端多代理深度审查 | 重要 PR 提交前 |
| `/team-onboarding` | 自动产出团队入门指南 | 新仓 / 新人 |
| `/insights` | 生成会话分析报告（涉及哪些区域、交互模式） | 季度复盘 |
| `/fewer-permission-prompts` | 扫只读工具调用并加到 settings 允许列表 | 减少弹窗骚扰 |
| `/claude-api [migrate\|managed-agents-onboard]` | 加载 Claude API 参考 / 升级代码到新模型 | 有 API 调用的项目 |

### 2.6 会让效率翻倍的"非命令"技巧

- **Esc 键**：方向跑偏立刻按 Esc 打断，比等它跑完再纠正便宜 10 倍。
- **双击 Esc**：回到本对话更早一条消息，从那里分叉（"读档"）。
- **Shift + Tab**：在六档 permission mode 间循环。
- **Ctrl + B**：把当前任务塞到后台运行（`/tasks` 看进度）。
- **Ctrl + T**：切换 Todo 列表显示。
- **拖入图片**：架构图、报错截图、Figma 截图直接拖进终端，CC 会识别。
- **多行输入**：粘长日志 / stack trace 前先开多行：
  - `\` + `Enter`（**所有终端通用**，最稳）
  - `Ctrl+J`（任意终端，无需配置）
  - `Shift+Enter`（iTerm2 / WezTerm / Ghostty / Kitty / Warp / Apple Terminal 原生支持；VS Code / Cursor 等需先 `/terminal-setup`）
  - `Option+Enter`（macOS 需先把 Option 设为 Meta）

## 3. Thinking 关键字（最被低估的能力）

任务描述里包含关键字，CC 会分配**更多内部推理 token**：

| 关键字 | 大致预算 | 强度 |
|--------|------|------|
| `think` | ~4,000 tokens | 低 |
| `think hard` / `megathink` | ~10,000 tokens | 中 |
| `think harder` / `think more` / `think a lot` | ~31,999 tokens | 高 |
| `ultrathink` | ~31,999 tokens（封顶）| 最高 |

> 这些关键字是 CC 客户端硬编码的预算映射；只在 Claude Code 里生效，在 claude.ai 网页或纯 API 调用里没有。**与 `/effort` 是两套机制**：thinking 关键字调"单回合推理深度"，effort 调"任务级努力档位"，可叠加。

**取舍法则**：

- 架构设计、难 bug 定位、并发/一致性问题 → `ultrathink` + `/effort xhigh`
- 普通重构、写单测、加日志 → 不加关键字，`/fast` 也可以
- 注意：思考越深越贵越慢；不是默认开越好

> 经验：**写代码的活用普通模式，想"怎么写"的活用 thinking 模式。**

## 4. Plan 模式 —— 资深工程师最该用起来的功能

进入方式（任选其一）：

- **Shift+Tab 循环到 plan**（最推荐）。
- `/plan [task description]`：临时切到 plan，可直接传任务。
- 启动时 `claude --permission-mode plan`：以 plan mode 直接开会话。
- 设置默认值：`settings.json` 加 `"defaultMode": "plan"`。

进入 plan mode 后：

- CC 只能 Read / Grep / Glob / WebSearch / TodoWrite 等只读工具。
- 产出一份结构化实施计划。
- 你 review 完，CC 调 `ExitPlanMode` 工具触发"切出 plan，开始执行"——你点 "Accept plan" 才会真的进 Agent 模式。

**什么任务必走 Plan 模式？**

1. 任何涉及 3 个以上文件或跨模块的改动
2. 有架构选型空间的（"加缓存"——Redis? 进程内? 文件? 这时该规划）
3. 你自己也没完全想清楚的需求
4. 对生产行为有影响的（迁移、回滚、schema 变更）

**反模式**：已经完全想清楚的一行代码改动也走 Plan 模式——纯纯浪费时间。

## 5. 推荐的"日更"工作流

实现新需求的典型流程：

```
1. /clear                          # 换干净上下文
2. 描述需求 + @相关目录 + "ultrathink + 进入 plan 模式"
3. CC 产出计划 → 你 review、纠偏、补充约束
4. Accept plan → Agent 自动执行
5. 执行过程中：
   - 看到偏了立刻 Esc
   - 需要补文档：@具体文件 追加提示
6. 让它 /review 自查
7. !pnpm test && !pnpm lint        # 直接 shell 跑验证
8. 让 CC 基于 diff 写一份 commit message 草稿
9. /cost 记录成本，/clear 准备下一个任务
```

查 Bug 的典型流程：

```
1. /clear
2. 贴报错日志 + 复现步骤 + "ultrathink, 先只分析不要改代码"
3. CC 输出若干根因假设 + 每个假设的验证方法
4. 你选一个最像的 → "按假设 2 继续验证，允许读代码、运行测试，但不要改代码"
5. 确认根因后："现在按最小改动修复，附带回归测试"
6. /review → 验证 → 提交
```

## 6. Headless & 脚本化

CC 除了交互式，还可以 `claude -p "<prompt>"` 以 **headless / non-interactive** 方式跑，适合：

- CI 里让它做自动化 code review
- pre-commit 钩子里检查敏感信息
- 批量跑同一个 prompt 处理上百个文件
- 每天早上给我总结昨天 PR 变更的 cron

常用参数：

```bash
# 基础
claude -p "检查这段 diff 有没有引入 N+1 查询" --output-format stream-json
claude -p "..." --allowedTools "Read,Grep,Bash(git diff:*)" --max-turns 5

# 极简启动（跳过 hooks/skills/plugins/MCP/auto memory/CLAUDE.md 自动发现）
# 脚本场景下显著快很多
claude --bare -p "..."

# 给预算护栏，超了直接退
claude -p --max-budget-usd 2.00 --max-turns 10 "..."

# 强制结构化产出，便于下游脚本解析
claude -p "..." --output-format json --json-schema ./schema.json

# 动态注入子代理定义（不需要落盘到 .claude/agents/）
claude -p "..." --agents '{"explorer":{"description":"...","tools":"Read,Grep"}}'
```

`--output-format` 三种：

| 值 | 适用 |
|----|------|
| `text`（默认） | 人看 |
| `json` | 简单脚本解析，整段一次性返回 |
| `stream-json` | 长任务 / 需要边跑边消费事件流 / CI 流式日志 |

**远端 / 多端的对照**（详见 06）：

- `claude --remote "<desc>"`：把会话开到云端 sandbox 跑（不占本机），适合"花两小时做一次大扫除"
- `claude --teleport`：把 claude.ai 网页上的会话拉回终端继续
- `claude --remote-control [name]`：从 claude.ai 远程控制本会话

**安全红线（headless / 自动化场景必读）**：

- 务必限制 `--allowedTools` 和 `--max-turns`，防止跑飞。
- **`--dangerously-skip-permissions`**（等同 `--permission-mode bypassPermissions`）会**跳过所有权限弹窗**——本地批量任务可考虑，**生产 / CI 严禁**。社区有过事故，等价于把仓库写权限给 prompt 注入者。
- CI 鉴权用 `claude setup-token` 生成长期 OAuth token，或用 API key（`ANTHROPIC_API_KEY`），不要复用个人订阅 session。
