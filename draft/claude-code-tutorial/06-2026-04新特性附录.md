# 06 · 2026/04 新特性附录

> 这一章是"过去半年新增 / 演化快"的特性集合。前 5 篇是稳定心智模型，本篇是**会过期**的事实清单——半年后请重新核对官方文档。

## 1. 模型族与 effort

| 模型 ID | 别名 | 默认 effort | 上限 | 拿来干什么 |
|---------|------|------------|------|-----------|
| `claude-opus-4-7` | `opus` | xhigh | max | 架构、复杂 bug、并发/一致性、ultrathink 场景 |
| `claude-sonnet-4-6` | `sonnet` | high | max | 日常实现、重构、写测试 |
| `claude-haiku-4-5-20251001` | `haiku` | — | 不支持 effort | 大批量 / 简单任务 / subagent 探索 |

**`/effort` 五档**：`low / medium / high / xhigh / max`。任务越复杂调越高，但 token 成本几乎线性涨。

**`/fast`**（仅 Opus 4.6 路径，跳过部分推理特效）：

- ✅ 适合：明确的实现任务、单元测试、改字符串、调样式
- ❌ 不适合：架构决策、根因定位、跨模块影响评估

**`opusplan` 混合策略**（最高性价比）：plan 走 Opus 4.7，execute 切 Sonnet 4.6。在 `settings.json`：

```json
{ "modelStrategy": "opusplan" }
```

## 2. Bundled skills 全景

官方在每个会话内置的 prompt 驱动 skill，`/help` 里 Purpose 列标 **Skill**。**这是提示词模板，不是硬编码逻辑**——所以行为会随版本升级而漂移，关键决策不要全押在 skill 输出上。

| Skill | 何时用 | ROI | 注意点 |
|-------|--------|-----|--------|
| `/batch <instruction>` | 大规模并行改动（跨包改名、批量迁移） | 高（省时间） | 费 token：5–30 个 worktree subagent 并发 |
| `/debug [description]` | 卡了不知道为什么、CC 行为反常 | 高 | 会开 debug 日志，结束记得关 |
| `/simplify [focus]` | 提交前自查重复实现、找重用机会 | 中 | 偏审查类，不会强改代码 |
| `/loop [interval] [prompt]` | 盯 CI、轮询任务状态 | 高（运维场景） | 持续花 token，记得加退出条件 |
| `/claude-api [migrate\|managed-agents-onboard]` | 项目里有 Claude API 调用 | 中 | 升级模型时省事 |

> 上表只列已在官方 docs 明确的 5 个。其余在社区/旧版本提到过的（如 `/security-review`、`/ultrareview`、`/ultraplan`、`/team-onboarding`、`/insights`、`/fewer-permission-prompts`）有的是真命令、有的可能是 plugin 提供——**用之前先 `/help` 看一下当前版本是否真的有**，文档没有的就当不存在。

## 3. 多端协同

| 端 | 进入方式 | 适用 |
|----|---------|------|
| 终端 CLI（最强） | `claude` | 日常主力 |
| VSCode / JetBrains 扩展 | IDE 插件 | 边写边问 |
| Cloud Web Session | `claude --remote "<task desc>"` | 长任务（数小时）丢云端不占本机 |
| Teleport | `claude --teleport` | 把 claude.ai 网页的会话拉回终端继续 |
| Remote Control | `claude --remote-control [name]`（也叫 `--rc`）；或 `claude remote-control` 起 server-only | 在路上用 claude.ai 远程操控本机 session |
| Chrome 集成 | `claude --chrome` | 让 CC 操作浏览器（自动化测试、抓页面） |

**典型组合**：

- 出差路上：手机 claude.ai → 看本机长跑任务进度 → 必要时下指令 = `--remote-control`
- 周末跑大扫除：`claude --remote "做一次 services/billing 全模块的 lint 修复"` → 周一回来看结果
- 写文档要看页面：`claude --chrome` → "打开 a.com，截图，提取主要文案"

## 4. Agent Teams

`--teammate-mode auto|in-process|tmux`：让多个 sub-Claude 作为"团队成员"协作（不是 subagent 那种隔离卸载，而是平级协作，互相能看见消息）。

- `auto`（默认）：CC 自己挑展示模式
- `in-process`：在同一个终端窗口里轮流显示
- `tmux`：每个 teammate 一个 tmux pane

**真实成本**：~7× 单 agent token 用量（多个模型并发跑且互相沟通）。**不要默认开**——只在确实需要多视角并发的少量大任务用。

> 旧文档里提到的 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 已不再需要——agent teams 已 GA，详见官方 `/en/agent-teams` 文档。

## 5. Auto Memory（v2.1.59+）完整指南

CC 自动维护的记忆系统，独立于 CLAUDE.md。详细文件结构和 type 模型见 [04 · 2.7-2.8 节](./04-大型代码库导航与CLAUDE.md体系.md#27-auto-memoryv2159和-claudemd-互补的自动记笔记)。

**最佳实践**：

1. **第一周末 review**：打开 `~/.claude/projects/<project>/memory/MEMORY.md`，删掉错误条目（CC 偶尔会记错）。
2. **`feedback` 类要看**：CC 会记录"你纠正过它的方式"。如果发现一条规则其实是误会，及时删——否则会被未来所有会话沿用。
3. **关：** `{ "autoMemoryEnabled": false }` 或 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1`。
4. **铁律不要靠 auto memory**——记忆是上下文，不是强约束。硬约束仍然进 CLAUDE.md。

## 6. Permission modes 六档

`Shift+Tab` 循环切换。

| 模式 | 编辑 | Bash | 何时用 |
|------|-----|------|--------|
| `default` | 弹窗 | 弹窗 | 不熟悉的仓库、第一次跑某个 plugin |
| `acceptEdits` | 自动 | 弹窗 | 主力日常 |
| `plan` | 不能改 | 不能跑 | 设计阶段、调研阶段 |
| `auto` | CC 按工具风险自判 | 同 | 中等熟悉的项目，又不想全弹 |
| `dontAsk` | 不问、按默认决策走 | 同 | 危险但不破墙；半信任的批量任务 |
| `bypassPermissions`（= `--dangerously-skip-permissions`） | 全跳 | 全跳 | **本地玩，生产/CI 严禁** |

CLI flag：`--permission-mode <mode>`，`--allow-dangerously-skip-permissions`（把 bypass 加入 Shift+Tab 循环但不直接进入）。

## 7. 常见反模式

| 反模式 | 后果 | 正确做法 |
|--------|------|---------|
| 默认开 `bypassPermissions` 跑生产/CI | 被 prompt injection 一次性接管仓库写权限 | CI 用 `--allowedTools` 白名单 + `--max-turns` |
| 把 agent teams 当默认 | 7× token 烧光配额 | 只在真正多视角并发时开，平时用 subagent |
| Bundled skill 与自定义同名 skill 共存 | 自定义优先，但你以为在用官方版本 | 自定义 skill 起独特名字（如 `team-debug`） |
| Skill 写"这次的步骤"而非"标准指令" | Skill 加载一次后常驻，下次还会按错的步骤跑 | Skill = 标准 SOP，一次性事项写在 prompt 里 |
| auto memory 当文档写 | 记忆膨胀、互相冲突 | auto memory 只记"看不出来的事"；规范进 CLAUDE.md |
| 项目级 `.mcp.json` 没人 review | 接入恶意 MCP 一键执行任意命令 | 第一次启用时 CC 弹批准；commit 前必须 PR review |
| `/effort max` + Opus 4.7 当默认 | 单任务 token 翻数倍 | 简单任务降到 medium 或切 Sonnet |
